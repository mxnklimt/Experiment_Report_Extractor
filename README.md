# experiment-report-extractor

一个 Claude Code Skill，通过迭代确认流程从实验数据中生成带有交互图表的单文件 HTML 汇报。每次生成前会逐一与用户确认：文件范围 → 重点指标 → 图表偏好 → 逐节审批，确保输出精准匹配用户意图。

## ✨ 功能特性

- 💬 **迭代确认流程**：分阶段与用户确认（文件范围 → 重点指标 → 图表偏好 → 逐节审批），不盲目生成
- 🤖 **AI 先行分析**：每次确认前 AI 先自行分析数据、提出结论假设，用户只需对齐/修正，无需从空白开始填写
- 📂 **多文件扫描**：自动遍历目录，识别所有 `.md` 实验文档 + `.csv`/`.json` 等数据文件
- 🔍 **正文/附录智能拆分**：用户挑选核心指标进入正文重点展示，其余列完整保留在文末附录
- 📊 **图表偏好确认**：自动建议图表类型后由用户确认，支持柱状图/折线图/散点图
- 🧪 **纯数据模式**：即使没有 `.md` 实验文档，也能从纯数据文件生成报告（用户补充背景信息）
- 📦 **零依赖输出**：生成单个独立 HTML 文件，ECharts 完整内联，离线打开即用
- 🎨 **现代卡片布局**：浅灰底 + 白色卡片 + 蓝色强调色，结论前置，字体与图表尺寸保证可读性
- ✋ **逐节审批**：生成过程中每节（全局结论 → 每卡片 → 附录）逐一呈现、逐一审批

## 📋 工作流程

Skill 通过 9 步迭代确认流程生成报告：

| 步骤 | 内容 | 确认点 |
|------|------|--------|
| 1 | 扫描目录/文件 | 确认报告范围 |
| 2 | 展示数据结构（列名、章节） | 确认数据理解正确 |
| 3 | 正文 vs 附录列选择、实验命名 | 用户挑选重点指标 |
| 4 | 图表类型、布局偏好 | 确认展示方式 |
| 5 | 汇总"报告蓝图" | **HARD-GATE — 最后确认后才开始生成** |
| 6 | 逐节生成正文 | 每节逐一审批 |
| 7 | 生成附录（全量数据） | 审批 |
| 8 | 整体自检 | AI 自动执行 |
| 9 | 用户最终审查 | 交付 `report.html` |

## 📁 数据文件关联规则

| 优先级 | 规则 |
|--------|------|
| 1 | 同名匹配：`exp1.md` 自动关联 `exp1.csv` / `exp1.json` 等 |
| 2 | 目录分组：子目录内的 `.md` 关联同目录下所有数据文件 |
| 3 | 手动指定：用户明确说明关联关系时优先采用 |

**支持格式**：`.csv` / `.tsv` / `.json` / `.jsonl` / `.xlsx`，格式模糊时 LLM 自动解析。

**图表类型自动选择**：

| 数据形态 | 图表类型 |
|----------|----------|
| 两列（索引 + 数值）| 折线图 |
| 一列类别 + 一列数值 | 柱状图 |
| 三列以上数值 | 多系列折线图 |
| 两列数值对 | 散点图 |
| Markdown 中注释 `<!-- chart: bar -->` | 按指定类型 |

## 🖥️ 输出效果

```
report.html
├── 页头：项目标题 + 统计条（实验数 / 图表数）
├── 实验卡片 #01
│   ├── 实验名称（来源文件名）
│   ├── 实验结果（文本 + 表格）
│   ├── 结论（黄色左边框高亮框）
│   └── ECharts 图表（如有数据文件）
├── 实验卡片 #02
│   └── ...
└── Footer
```

- 图表高度 ≥ 400px，所有字体 ≥ 16px
- X 轴标签始终水平展示，禁止旋转
- 一指标一图原则，每个图表只回答一个对比问题
- ECharts 完整源码内联，无 CDN 依赖，离线可用

## 🚀 使用方式

在 Claude Code 中直接用自然语言触发：

```
帮我处理 /path/to/experiments/ 目录，生成实验汇报
```

Claude 会通过迭代确认流程：
1. 扫描目录中所有 `.md` 和数据文件 → 与你确认报告范围
2. 展示每个数据文件的所有列 → 你挑选正文重点展示的列（其余进附录）
3. **AI 先分析数据，提出核心结论假设** → 你只需确认或修正，无需自己想结论
4. 确认实验命名、分组标签、图表类型偏好
5. 呈现"报告蓝图"（含 AI 拟定的全局结论）供你最终审批 → 批准后开始生成
6. 逐节生成（AI 先提各节结论 → 你确认 → 全局结论 → 每卡片 → 附录）→ 每节等你审批后继续
7. 整体自检 → 你最终审查 → 交付 `report.html`

## 📐 设计约束（Hard Rules）

以下规则在技能文档中强制执行，Claude 不会因用户要求而违反：

| 规则 | 原因 |
|------|------|
| 图表库只用 ECharts，禁止 Chart.js / Plotly | 一致性与离线支持 |
| ECharts 必须内联，禁止 CDN 引用 | 保证文件零外部依赖 |
| 不捏造或推算数据文件中不存在的数值 | 实验报告的数据诚实性 |
| 图表字体 ≥ 16px，高度 ≥ 400px | 保证可读性 |
| **生成前必须先确认报告蓝图** | 确保用户意图和输出一致 |
| **正文只展示用户选定的重点列** | 正文精简、重点明确 |
| **所有被过滤的列必须在附录中完整呈现** | 数据完整性 |
| **AI 先分析数据提出结论，用户对齐修正** | 减少用户认知负担，AI 承担分析工作 |
